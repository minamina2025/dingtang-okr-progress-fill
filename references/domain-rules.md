# 领域规则和个人规则

公司通用 OKR 进展填写 skill 必须保持领域中立。招聘、销售、研发、客服、产品、运营等业务口径只能作为可选领域规则或个人规则加载，不能写死在主流程里。

## 加载顺序

判断 KR 进度口径时，按下面顺序寻找规则：

1. 线上 KR 明文写出的目标值、结果指标、周期、验收标准。
2. 公司发布的 OKR 规则、领域规则或指标定义。
3. 相关业务系统中当前用户有权限读取的指标定义、字段说明或 Dashboard 口径。
4. 当前用户 Memory 或个人配置中已经确认的个人/岗位规则。
5. 通用量化/定性判断。

如果规则冲突，使用同样的优先级。个人规则只能影响当前用户自己的草稿，不能作为公司全员默认规则。

## 规则形态

领域规则至少要说明：

- `domain`：领域，例如招聘、销售、研发、客服、产品、运营。
- `appliesWhen`：什么 KR 文本或场景适用。
- `formula`：如何计算百分比；如果不能量化，说明为什么只能定性估算。
- `periodRule`：如何处理季度、月份、跨周期累计、历史遗留项。
- `evidenceSources`：可用证据源和权限边界。
- `presentationPolicy`：哪些内容可以展示给管理者，哪些必须脱敏或只保留权限链接。

## 使用要求

- 能直接按实际进展计算百分比时，使用量化公式，并在 `calculationBasis` 写清分子、分母、周期和口径来源。
- 只有 KR 不够量化、过于定性，或单一交付物没有自然分子分母时，才使用通用进度百分比参考。
- 业务系统 Dashboard 的默认比例不一定等于 OKR 进度。必须确认它和 KR 承诺的结果指标一致，不能一致时只能作为辅助证据。
- 没有权限读取的证据不得猜测。进展说明中写明权限缺口或证据缺口即可。
- 领域规则不得要求读取用户无权限的数据，也不得要求暴露薪资、绩效、候选人姓名、客户敏感商业条款、私人联系方式、原始评价等敏感信息。

## 个人规则示例

个人规则适合放在用户 Memory、私有配置文件或个人领域 skill 中。例如某位 HR 负责人可能定义招聘 KR 的季度口径；某位销售负责人可能定义回款和签约的优先级；某位研发负责人可能定义 Story Points 和上线验收的折算方式。

通用 skill 只负责发现并引用这些个人规则，不把它们写成所有人的默认规则。
